# symgov UI design artifacts

This folder contains a lightweight update package for the two core experiences:

- `design-briefs.md` - concise design briefs for Standards View and Workspace View, plus assumptions and follow-ups
- `wireframes.md` - desktop and mobile ASCII wireframes for both views
- `component-list.md` - shared components, per-view layout grids, and interaction priorities
- `svg-rendering-patterns.md` - guidance for inline SVG, sprite usage, lazy loading, and published-page rendering rules
- `prototype-representative.html` - small representative prototype for Standards View with inline SVG symbol cards and contextual chat
- `prototype-representative.css` - minimal styles for the representative prototype

Scope for this pass:

- Standards View uses a split browse/detail layout with integrated clarification chat tied to the selected symbol or published page.
- Workspace View focuses on bulk triage, change queue handling, compare/approval tools, and fast review throughput.
- Published pages expose only the latest approved revision.
